# 決定ログ(Decision Log)

チームが出した主要な見立てを記録します。後で `retrospective-reviewer` が実際の結果と照合し、
精度(キャリブレーション)を評価して `LESSONS.md` に教訓を蓄積するための土台です。

## 記録フォーマット
```
## YYYY-MM-DD
- 〔対象(銘柄/指数/テーマ)〕: 〔見立て〕 | 確信度: 高/中/低 | 反証条件: 〜 | 検証予定: YYYY-MM-DD頃
  - 結果(後日追記): 的中/外れ/保留 — 実際にどうなったか / 論理は正しかったか
```

## 重要な考え方
- 「当たった/外れた」だけでなく「論理(プロセス)が正しかったか」を分けて記録する。
- 確信度を必ず付ける。後で「高確信の的中率」を測りキャリブレーションを是正するため。
