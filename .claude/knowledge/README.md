# 知識ベース（`.claude/knowledge/`）

このフォルダは、**セッションをまたいで共有されるチーム共通の知識ベース**です。
ここに置いたファイルは git で管理され、`CLAUDE.md` から参照されているため、
**新しいセッションでも自動的に前提知識として読み込まれます**（毎回リポジトリがクローンされるため）。

## 仕組み（なぜ他セッションでも共有されるのか）

1. Claude Code on the web はセッション開始時にリポジトリを新規クローンする。
2. `CLAUDE.md` は全セッションで自動的に読み込まれる。
3. `CLAUDE.md` がこのフォルダのプレイブックを「必読」として参照している。
4. → **commit & push した知識は、以降の全セッションに自動共有される。**

知識を増やしたいときは「学習して」と依頼すれば、内容を整理してここに追記・commit します。

## 関連する共有の仕組み

- `../../CLAUDE.md` … 事業前提・司令塔の役割・専門家一覧・共通の判断軸（全セッション自動ロード）
- `../agents/` … 専門家サブエージェントの定義
- `../commands/日報.md` … `/日報` でセッションログを生成する仕組み
- `../../docs/日報/` … セッションごとの日報（履歴・引き継ぎ）

## 追加・更新のルール

- 1テーマ＝1ファイル。重複する内容は新規ファイルを作らず既存に追記する。
- 出典が同じ資料の場合は、新情報だけを既存プレイブックに反映する。
- 法規・税務・倫理に関わる数値や主張には「要専門家確認」「仮説」等の但し書きを添える。
- 特定業者への宣伝・誘導・連絡先は恒久知識に含めない（一般化して記載）。
