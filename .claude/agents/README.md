# Subagents（専門サブエージェント）

このディレクトリには、Claude Code から呼び出せる**専門特化したサブエージェント**を定義しています。
サブエージェントは独立した文脈で動き、結論だけを親エージェントに返すため、メインの会話を汚さずに
重い調査やレビューを並列で任せられます。

## 入っているサブエージェント

| 名前 | 役割 | 使うツール |
|------|------|-----------|
| `code-reviewer` | 変更差分のレビュー（バグ・可読性・テスト漏れ） | Read, Grep, Glob, Bash |
| `repo-explorer` | リポジトリ横断の読み取り専用調査 | Read, Grep, Glob, Bash |

## 使い方

自然文で依頼すると、Claude Code が `description` を見て自動的に該当サブエージェントを選びます。

```
このブランチの変更をレビューして
README の生成処理はどこで実装されてる？
```

明示的に指名することもできます。

```
code-reviewer サブエージェントで差分を確認して
```

## ファイル形式

各サブエージェントは Markdown + フロントマターで定義します。

```markdown
---
name: エージェント名（小文字・ハイフン）
description: いつ呼ばれるべきかを書く。これを見て自動選択される
tools: 使えるツールをカンマ区切りで（省略すると全ツール）
model: sonnet / opus / haiku（省略可）
---

ここにサブエージェントへのシステムプロンプト（指示）を書く。
```

## 新しいサブエージェントを足すには

1. このディレクトリに `<名前>.md` を作る。
2. フロントマターに `name` と `description` を必ず書く（`description` が自動選択の決め手）。
3. `tools` で権限を最小限に絞ると安全（例: 調査用なら書き込みツールを外す）。

> 詳細は公式ドキュメント: https://code.claude.com/docs/en/sub-agents
