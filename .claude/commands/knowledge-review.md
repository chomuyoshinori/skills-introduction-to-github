書籍知識化チームの `knowledge-retrospective` を使って、知識の定期レビュー(学習ループ)を実行してください。

## 入力
- 対象: `$ARGUMENTS`(書名を指定すればそのノートを集中レビュー)。指定がなければ、`knowledge/INDEX.md` で**次回レビュー予定を過ぎているノート**を対象にする。

## 手順
1. `KNOWLEDGE_PROFILE.md` と `KNOWLEDGE_LESSONS.md`(あれば)を読む。
2. `knowledge/INDEX.md` を読み、レビュー対象ノートを決める(台帳が無ければ `knowledge/*.md` を走査して台帳を新規作成)。
3. `knowledge-retrospective` を起動し、対象ノートに対して:
   - **想起テスト**(間隔反復): 本を見ずに思い出す形式のQ&Aを出題(模範解答付き)。
   - **鮮度チェック**: 重要な事実・数値・固有名詞をWebで再確認し、古い記述にフラグ。
4. 結果を `knowledge/reviews/YYYY-MM-DD.md`(学習日報)に保存。
5. `knowledge/INDEX.md` の「最終レビュー日 / 次回レビュー予定 / 想起スコア」を更新。
6. 気づいたコツを `KNOWLEDGE_LESSONS.md` に追記。
7. チャットには「今日の出題(本を見ずに答えてみる用)」と「更新が必要になった知識」を要約する。

## 間隔反復スケジュール
新規作成日を起点に 1日 → 1週 → 1ヶ月 → 3ヶ月 → 6ヶ月 →(以降半年ごと)。
想起結果が ✗(忘れた)なら間隔を1つ戻し、◎が続けば延ばす。

## 注意
- 想起テストは答えを先に見せず、本人が思い出せるかを測る設計にする。
- 鮮度チェックでの更新は出典・時点を必ず残す。未確認は断定しない。
