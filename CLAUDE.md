# 日本株 リサーチ・分析チーム

毎日の日本投資情報をまとめ、中期・成長重視(中リスク)の投資家向けに分析レポートを作成する
エージェントチームです。批判的検証と継続的な学習・改善を組み込んでいます。

> ⚠️ **重要**: 本リポジトリのレポート・分析・コンサルはすべて情報整理および教育目的であり、
> 投資助言(金融商品取引法上の投資助言・代理業)ではありません。将来のリターンを保証するものでも
> ありません。最終的な投資判断はご自身の責任で行ってください。

## チーム構成(`.claude/agents/`)
| エージェント | 役割 |
|---|---|
| `macro-market-analyst` | マーケット概況(指数・為替・金利・海外市場・金融政策) |
| `news-curator` | 当日の重要ニュースを重要度順に要約 |
| `sector-theme-analyst` | セクター・テーマの資金フローと中期の追い風/逆風 |
| `equity-analyst` | 日本の個別株の財務・成長性・バリュエーション・リスク分析 |
| `us-equity-analyst` | 米国個別株(株式枠・中リスク統合)。ドル建て・米国会計・為替・米国課税を考慮 |
| `crypto-analyst` | 暗号資産(**別枠「高リスク」**)の時価総額・トークノミクス・アンロック・規制・サイクル分析 |
| `red-team-critic` | **批判的検証(レッドチーム)**。強気論に反対シナリオをぶつけ、バイアス・見落とし・データの弱さを突く検証ゲート |
| `retrospective-reviewer` | **学習・改善ループ**。過去の見立てを実績と照合し、教訓を `LESSONS.md` に蓄積 |
| `report-editor` | 編集長。全出力を統合し、レッドチーム検証を必ず通して日次レポートに仕上げる司令塔 |

## ワークフロー
```
INVESTMENT_PROFILE.md + LESSONS.md を踏まえる
        │
        ▼
 4アナリスト(マクロ/ニュース/セクター/個別株)が並行リサーチ
        │
        ▼
 red-team-critic が全出力を批判的に検証（必須ゲート）
        │
        ▼
 report-editor が統合 → reports/YYYY-MM-DD.md
        │              → DECISION_LOG.md に見立てを記録
        ▼
 （週次/月次）retrospective-reviewer が実績と照合 → LESSONS.md 更新
```

## 使い方(スラッシュコマンド)
- `/daily-report` — 本日の日次レポートをチーム総出で作成。`/daily-report 6758 ソニーG` のように銘柄指定も可。
- `/crypto-report` — 暗号資産(高リスク枠)のリサーチレポートを作成。`/crypto-report BTC ETH` のように銘柄指定も可。
- `/retrospective` — 過去の見立てを実績と照合し、精度評価と教訓蓄積を行う。

## 毎日の自動実行(GitHub Actions)
`.github/workflows/auto-research-reports.yml` が cron で各レポートを自動生成・コミットする。
- 日本株: 平日 15:00 JST(大引け後) / 米国株: 平日 翌07:00 JST(米引け後) / 暗号資産: 毎日 08:00 JST
- **有効化の条件**: ①このワークフローを **main にマージ**(cronはデフォルトブランチでのみ発火) ②リポジトリ Secrets に **`ANTHROPIC_API_KEY`** を登録。
- 手動テストは Actions タブの「Run workflow」(対象 jp/us/crypto を選択)。
- ⚠️ サブエージェント多数+Web検索のため実行ごとに API 利用料が発生する。

## 資産クラスとリスク枠
- **株式**: 中期・成長重視・**中リスク**(`INVESTMENT_PROFILE.md`)。
- **暗号資産**: 株式とは**分離した「高リスク」枠**(`CRYPTO_PROFILE.md`)。同じ中リスク基準を流用しない。

## 主要ファイル
- `INVESTMENT_PROFILE.md` — 株式の投資家プロファイル(スタイル・リスク・保有銘柄・注目テーマ)。**ここを更新すると分析の方向が変わる。**
- `CRYPTO_PROFILE.md` — 暗号資産プロファイル(高リスク枠・保有銘柄)。
- `DECISION_LOG.md` — 出した見立ての記録(後の検証用)。
- `LESSONS.md` — 蓄積された教訓。全アナリストが作業前に参照する。
- `reports/` — 日次レポート保存先。

## 設計思想
1. **分業**: マクロ・個別・セクター・ニュースを専門化し、深さと網羅を両立。
2. **批判を制度化**: レッドチームを必須ゲートにし、強気論への流されを防ぐ。確信度と反証条件を必ず付す。
3. **学習し続ける**: 見立てを記録し実績と照合、キャリブレーションを是正して精度を上げ続ける。

# 書籍知識化チーム(投資チームとは別系統)

書籍(PDF/テキスト/抜粋)を「わかりやすく構造化された知識ノート」に変換する高品質チーム。
ローカルCLIで自分の書籍フォルダを順に知識化する用途を想定。

## チーム構成(`.claude/agents/knowledge-*`)
| エージェント | 役割 |
|---|---|
| `knowledge-lead` | 編集長(リーダー)。全役割を統合し1冊分の知識ノートに仕上げる司令塔 |
| `knowledge-teacher` | **中核「教師」**。正確な読解・分析評価・Web補完・わかりやすい説明 |
| `knowledge-critic` | 批判的な副リーダー。読み違い・要約の歪み・本の主張への反証を突く検証ゲート(必須) |
| `knowledge-engineer` | 企画/エンジニア。知識の構造設計・整形・用語集/チェックリスト等の部品化 |
| `knowledge-art-director` | アートディレクター。Mermaid/表で概念図・関係図・フローを作図 |
| `knowledge-ethicist` | 倫理。著作権・引用の妥当性・偏り・誤情報・断定の行き過ぎを点検(必須ゲート) |

## ワークフロー
```
KNOWLEDGE_PROFILE.md + LESSONS.md を踏まえる
   → teacher が読解・分析・Web裏取り
   → critic が批判的検証(必須ゲート)
   → engineer が構造化 / art-director が図解
   → ethicist が倫理・著作権チェック(必須ゲート)
   → lead が統合 → knowledge/<書名>.md
```

## 使い方
- `/knowledge` — 対象書籍を知識化。`/knowledge books/part1.pdf` のようにパス指定可。
- 書籍が複数パートに分割で届く場合は、**全パートが揃ってから**統合する。

## 著作権・運用ルール
- 書籍本文の丸写し・大量転記をしない(要約・再構成のみ。引用は短く・出典明記)。
- 市販書籍のPDF本体は **コミットしない**(`books/` と `*.pdf` 等は `.gitignore` 済み)。成果物の `knowledge/*.md` のみ管理。
- 成果物の公開は私的利用の範囲に留める想定。倫理チェックを通っていないノートは保存・公開しない。

## 主要ファイル(知識化)
- `KNOWLEDGE_PROFILE.md` — 読者像・用途・深さ・形式の設定。**ここを更新するとノートの質感が変わる。**
- `knowledge/` — 完成した知識ノートの保存先。
- `books/` — 元のPDF/原稿置き場(Git管理外)。
